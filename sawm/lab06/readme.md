# Лабораторная работа №6. Журналирование действий пользователей в целях безопасности(logging)

## Требования к лабораторной работе

1. Используйте ранее созданное приложение в 4-5-й лабораторных работах.
2. Создайте файл логов для хранения в нем действий пользователей: кто, когда вошел, какие
действия выполнил по отношении с приложением, ... когда вышел. (посмотрите пример с
лекции для добавлени и других деталей).
3. Добавьте пользователю «администратор» опцию, которая доступна ему только после
авторизации и проверки роли, через которую он может посмотреть содержимое лог-файла.
4. Объясните преподавателю необходимость регистрации действий пользователей,
необходимость хранения обезличенных, неконфиденциальных данных в файле журнала,
необходимость анализа содержимого файлов журнала, а также ротацию/изменение
журналов.

- Необходимость регистрации действий пользователей:  
  Позволяет отслеживать действия для безопасности, обнаружения нарушений и аудита системы.

- Необходимость хранения обезличенных, неконфиденциальных данных в файле журнала:  
  Обеспечивает возможность анализа работы системы без риска раскрытия личной информации пользователей.

- Необходимость анализа содержимого файлов журнала:  
  Помогает выявлять ошибки, сбои, подозрительные действия и оптимизировать работу приложения.

- Необходимость ротации/изменения журналов:  
  Предотвращает переполнение диска, упрощает управление логами и поддерживает производительность системы.
