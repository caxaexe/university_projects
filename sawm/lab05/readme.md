# Лабораторная работа №5. Контроль доступа к веб приложению

## Требования к лабораторной работе
1. Используйте мини-приложение созданную в 1-ой лабораторной и продемонстрируйте что
если пользователь знает URL ресурса, содержащий пано администрирования, может сразу
зайти туда, без аутентификации.
2. Внедрите правильно принципы использования сеcсии в веб приложение и докажите что
больше невозможно зайти в пано администрирования без успешной аутентификации –
аутентификация обязательна;
3. Дополнительно создайте аккаунт для пользователя «менеджер» (добавьте а таблицу «user»
дополнительное поле «роль» - у вас сейчас в поле роль должны быть 2 возможных значения
– «администратор» и «менеджер»). Пользователь с ролью «менеджер» должен иметь доступ
к другому пространству для работы и другим функционалом (пусть будут 2 функции доступны
этому виду пользователя), к которому он имеет доступ, также, после успешной
аутентификации.
4. Дополните БД необходимыми таблицами так чтобы была возможность авторизации
пользователей (используя переменные сесcии), так чтобы после аутентификации
пользователя, в зависимости от его роли он был перенаправлен на то пано (страницу) которая
должна быть ему доступна, соответственно его роли.
5. Проверьте, имеют ли пользователи доступ к разрешенным функциям и правильно ли
реализован контроль доступа на основе ролей и разрешений. Внедрите функции для
«менеджера» по просмотру данных учетных записей, изменению данных или их удалению,
но эти функции не должны быть доступны простому пользователю.
6. С обеих закрытых пространств должна существовать возможность выхода со «сбросом» на
страницу аутентификации.
7. Продемонстрируйте преподавателю, как работать с приложением для пользователей с
разными ролями, пройдя весь цикл событий для пользователей обеих ролей.
